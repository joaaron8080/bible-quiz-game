# PRD: 성경 퀴즈 게임 (Bible Quiz Game)

## Problem Statement

성경 지식을 재미있고 체계적으로 학습하고 싶은 사용자가, 구약과 신약을 아우르는 단계별 퀴즈 게임을 통해 실력을 점진적으로 키울 수 있는 웹 애플리케이션이 없다. 기존 성경 퀴즈 앱은 난이도 구분이 없거나 진행 상황이 저장되지 않아 지속적인 학습 동기 부여가 어렵다.

## Solution

구약→신약, 쉬움→어려움 순서로 설계된 10단계 성경 퀴즈 게임 웹앱. 4지선다형 문제, O/X 시각/청각 피드백, localStorage 기반 진행 저장, 단계 완료 축하 메시지를 제공한다. Next.js로 구현하여 Vercel에 배포한다.

## User Stories

1. 사용자로서, 1단계부터 10단계까지 순서대로 퀴즈를 풀고 싶다 — 체계적으로 성경 지식을 쌓기 위해.
2. 사용자로서, 각 단계에서 10개의 4지선다 문제를 풀고 싶다 — 집중적인 학습 세션을 갖기 위해.
3. 사용자로서, 문제를 선택하면 즉시 O/X 피드백을 받고 싶다 — 정답 여부를 바로 확인하기 위해.
4. 사용자로서, 정답 시 밝은 효과음, 오답 시 버저 효과음을 듣고 싶다 — 청각적 피드백으로 학습 효과를 높이기 위해.
5. 사용자로서, O/X 피드백이 전체 화면에 크게 표시된 후 자동으로 다음 문제로 넘어가길 원한다 — 흐름이 끊기지 않는 게임 경험을 위해.
6. 사용자로서, 10문제 중 7개 이상 맞추면 단계를 통과하고 싶다 — 적절한 성취감을 느끼기 위해.
7. 사용자로서, 단계 완료 시 축하 메시지와 팡파레 사운드를 보고 싶다 — 성취를 축하받기 위해.
8. 사용자로서, 축하 화면에서 다음 단계로 바로 이동할 수 있고 싶다 — 학습 흐름을 이어가기 위해.
9. 사용자로서, 단계를 통과하지 못해도 현재 단계를 재도전할 수 있고 싶다 — 포기하지 않고 다시 시도하기 위해.
10. 사용자로서, 재도전 시 새로운 랜덤 문제 10개가 출제되길 원한다 — 암기가 아닌 진짜 실력으로 통과하기 위해.
11. 사용자로서, 게임을 종료하고 다음에 다시 접속해도 내가 도달한 단계가 기억되길 원한다 — 처음부터 다시 시작하지 않기 위해.
12. 사용자로서, 앱 재접속 시 저장된 다음 단계에서 바로 시작하길 원한다 — 연속된 학습 경험을 위해.
13. 사용자로서, 진행 상황을 초기화하고 싶을 때 설정에서 리셋할 수 있고 싶다 — 처음부터 다시 시작하기 위해.
14. 사용자로서, 1-3단계에서 구약의 유명한 사건(창세기, 출애굽기, 다윗 등) 문제를 풀고 싶다 — 기초부터 시작하기 위해.
15. 사용자로서, 4-6단계에서 신약의 유명한 사건(복음서, 사도행전) 문제를 풀고 싶다 — 중급 지식을 테스트하기 위해.
16. 사용자로서, 7-8단계에서 구약 소선지서와 역사서의 세부 내용 문제를 풀고 싶다 — 심화 학습을 위해.
17. 사용자로서, 9-10단계에서 신약 서신서와 요한계시록의 세부 내용 문제를 풀고 싶다 — 최고 수준의 성경 지식을 검증하기 위해.
18. 사용자로서, 현재 몇 단계인지, 몇 번째 문제인지 항상 확인할 수 있고 싶다 — 진행 상황을 파악하기 위해.
19. 사용자로서, 단계 내 점수(몇 개 맞혔는지)를 실시간으로 볼 수 있고 싶다 — 현재 통과 여부를 가늠하기 위해.
20. 사용자로서, 10단계까지 모두 완료하면 최종 완료 메시지를 보고 싶다 — 모든 과정을 마친 성취감을 느끼기 위해.
21. 사용자로서, 모바일과 데스크톱 모두에서 편리하게 사용하고 싶다 — 어떤 기기에서든 학습하기 위해.
22. 사용자로서, 골드/크림 팔레트의 차분하고 품위 있는 UI를 보고 싶다 — 성경의 분위기에 맞는 학습 환경을 위해.

## Implementation Decisions

### 기술 스택
- **Framework**: Next.js (App Router) + TypeScript
- **Styling**: Tailwind CSS
- **배포**: Vercel (static export 또는 SSG)
- **상태 저장**: localStorage (백엔드 없음)

### 문제 데이터
- 총 300문제 (레벨당 30문제 풀, 매 도전마다 10개 랜덤 추출)
- 한국어 문제/보기/정답 해설
- 각 문제 구조: `{ id, level, question, options: [A,B,C,D], answer, explanation }`
- 정적 JSON 파일로 번들에 포함 (API 호출 없음)

### 난이도 구조
| 단계 | 범위 | 난이도 |
|------|------|--------|
| 1-3 | 구약 유명 사건 (창세기, 출애굽기, 다윗/솔로몬) | 쉬움 |
| 4-6 | 신약 유명 사건 (사복음서, 사도행전) | 중간 |
| 7-8 | 구약 소선지서, 역사서 세부 내용 | 어려움 |
| 9-10 | 신약 서신서, 요한계시록 세부 내용 | 매우 어려움 |

### 레벨 통과 기준
- 10문제 중 7개 이상 정답 (70%) → 통과
- 미통과 시: 현재 레벨 재시작, 풀(30개)에서 새 10개 랜덤 추출

### 게임 흐름 상태 머신
```
HOME → LEVEL_INTRO → QUESTION → FEEDBACK(0.8s) → QUESTION
                                               → LEVEL_RESULT(pass/fail)
                                                   → CELEBRATION → NEXT_LEVEL
                                                   → RETRY → LEVEL_INTRO
HOME → (저장된 레벨 있으면) LEVEL_INTRO(saved_level)
```

### 저장 스키마 (localStorage)
```typescript
interface GameProgress {
  highestLevel: number;      // 1-10, 완료한 최고 레벨
  currentLevel: number;      // 다음 시작할 레벨
  completedLevels: number[]; // 통과한 레벨 목록
}
```

### 사운드
- Web Audio API로 코드 생성 (외부 파일 없음)
- 정답: 밝은 3음 상승 멜로디 (C5 → E5 → G5)
- 오답: 낮은 버저음 (Bb3, 0.3초)
- 레벨 완료: 팡파레 (5음 상승 시퀀스)
- 최종 완료: 확장된 팡파레

### O/X 피드백
- 보기 선택 즉시 전체 화면 오버레이
- O (초록): 0.8초 후 자동으로 다음 문제
- X (빨강): 0.8초 후 자동으로 다음 문제 (정답 표시 포함)
- 오버레이 중 다른 보기 선택 불가

### 진행 저장 시점
- 레벨 통과 즉시 저장
- 앱 재접속 시 `currentLevel`에서 자동 시작
- 설정 화면에서 "진행 초기화" 버튼으로 리셋

## Testing Decisions

### 좋은 테스트 기준
- 구현 세부사항이 아닌 외부 동작 테스트
- 사용자 관점: "7개 맞추면 통과한다", "localStorage에 레벨이 저장된다"
- 컴포넌트 내부 state가 아닌 화면에 보이는 결과 검증

### 테스트 대상 모듈
- **레벨 통과 로직**: 7/10 경계값 (6개=실패, 7개=통과)
- **문제 랜덤 추출**: 30개 풀에서 10개, 중복 없음
- **localStorage 저장/복원**: 진행 상황 저장 및 재접속 복원
- **사운드 유틸**: Web Audio API 호출 여부 (mock)
- **게임 흐름**: HOME → QUESTION → FEEDBACK → RESULT 상태 전이

### 테스트 도구
- Jest + React Testing Library
- localStorage mock (`jest-localstorage-mock`)

## Out of Scope

- 사용자 인증 / 로그인
- 기기 간 진행 상황 동기화
- 멀티 프로필 (닉네임 분리)
- 문제당 타이머
- 리더보드 / 랭킹
- 소셜 공유 기능
- 관리자 페이지 (문제 추가/편집)
- 다국어 지원 (영어 등)
- 오프라인 PWA

## Further Notes

- 문제 데이터(300개)는 Claude가 생성하여 정적 JSON으로 번들에 포함
- 성경 번역 기준: 개역개정 (한국 기독교 표준)
- Vercel 배포 시 `next export` 또는 `output: 'export'` 설정 필요
- 향후 확장 가능 포인트: 챌린지 모드(타이머), 오답 복습 모드, 구절 암송 모드
